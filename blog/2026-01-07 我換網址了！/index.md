## 為什麼換？

原本用 GitHub 預設網址時，網址後面一定要帶你的專案名稱，還有尷尬的使用者名稱在前面 `/kaiblog`，這在設定網站的時候其實挺煩的。有時候圖片路徑沒抓好，或者是設定一些靜態工具時，那個斜槓（/）的位置沒弄對，圖片就直接死給你看。

我一直想找一個更短、看起比較帥的網域，但是要免費！！！

剛好看到 is-a.dev 這個東西。

## is-a.dev 是什麼？
這是一個專門給開發者的免費網域專案，簡單說就是讓你選一個名字掛在 .is-a.dev 前面。

它的申請方式很有趣，不是去什麼官網填表單，而是要去他們的 GitHub 倉庫 發一個 Pull Request。

你要在那邊新增一個小檔案，寫好你的名字跟網站位置。等對方的機器人檢查過、管理員點頭之後，網域就生效了。

**比起去那種廣告一堆的域名後台點來點去，這種解決的方式直覺多了。**

## 搬家過程的碎碎念
搬家這件事說難不難，但雜事真的很多：
 * 申請等了兩天，不過至少先搶到了！大家快搶！！！
 * 處理路徑問題： 以前文章網址前面都要掛個 /blog，現在直接移到最頂層。我把所有內部連結都整理了一下，現在結構乾淨多了，以後如果要再換網域也會省事很多。
 * 舊連結跳轉： 雖然換了新家，但舊家那邊我還是留了一個跳轉腳本。如果你是點到以前的舊連結，它應該會自動把你帶到新網址的對應位置。
 * GitHub 的設定： 在倉庫裡設定好網域後，GitHub 就會自動處理跳轉，這點倒是挺方便的。
 * 自動部署： 流程一樣是走 GitHub Actions，只是現在改用自定義網域模式。
